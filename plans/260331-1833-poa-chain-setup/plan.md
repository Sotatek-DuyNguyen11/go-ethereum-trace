**Status:** Ready for Implementation
**Branch:** master

## Overview

Setup 3-node Clique PoA private chain chạy bằng Docker Compose. Clique engine verification logic có sẵn trong `consensus/clique/`, nhưng **sealing bị disable + RPC APIs bị xóa** trong recent upstream commits. Cần restore trước khi setup Docker.

## ⚠️ Critical Finding

Upstream geth đã disable Clique mining (commit `c6823c77a`) và xóa RPC APIs (commit `f21adaf24`). Cần restore:

1. **`Seal()` panics** — thay vì sign block, nó panic. Code gốc đã bị xóa.
2. **`api.go` bị xóa hoàn toàn** — 235 LOC chứa `clique_getSigners`, `clique_propose`, etc.
3. **`CreateConsensusEngine()` yêu cầu TTD** — phải set `TerminalTotalDifficulty` = MaxInt64 (KHÔNG = 0).
4. **`Authorize()` thiếu `signFn`** — chỉ nhận address, không nhận signing function.
5. **`--unlock` flag deprecated** — has no effect (cmd/geth/main.go:344). Cần flag mới `--clique.signerkey`.
6. **`static-nodes.json` deprecated & ignored** — (node/config.go:448). Dùng `--bootnodes` hoặc `P2P.StaticNodes` trong config.toml.
7. **Signer key và node key là hai artifact khác nhau** — `signerkey` dùng để ký block Clique, `nodekey` dùng để tạo `enode` cho devp2p.

### Giải pháp

- Set `TerminalTotalDifficulty = math.MaxInt64` → chain luôn ở pre-merge → Beacon delegate mọi thứ cho Clique
- Restore `Seal()` với signing logic (dùng `crypto.Sign` trực tiếp thay vì `accounts.SignerFn`)
- Restore `api.go` cho RPC endpoints
- Thêm `signFn` field vào Clique struct
- Thêm `--clique.signerkey <path>` flag thay thế `--unlock`
- Init script phải generate **2 loại key riêng** cho mỗi node:
  - `signerkey` raw secp256k1 cho Clique sealing
  - `nodekey` cho P2P identity và `enode`
- Peer discovery qua `--bootnodes` enode URLs (không dùng static-nodes.json)
